Calculator Follow-Up Workflow

Automated behavior-based follow-up via GoHighLevel — triggered by PDF downloads and Home Depot link clicks
Date: April 14, 2026 Status: Code Complete — GHL Setup Pending Applies to: All 5 Calculators

Overview

This document describes the two automated follow-up workflows built for the Utah REIA calculator suite. The goal is to send relevant content to members based on what they are actively doing inside the calculators — without any manual outreach required.

The logic in plain English:
If a member downloads a deal report PDF → send them the Home Depot Pro resource.
If they then click the Home Depot link → send them all the calculator tools + AI Rehab Estimator.

What Was Built (Technical)

Two things were changed in the code. You do not need to touch the code — this section is for reference only.

1. New Vercel API Endpoints

Two new serverless functions were created — one for each Vercel project:

Calculators app
Calculators/api/track-event.js

Used by: Fix & Flip, BRRRR, Rental, Short-Term
Wholesale app
api/track-event.js

Used by: Wholesale Calculator

These endpoints receive the event from the calculator and forward it to the GHL webhook. They require one environment variable to be set in Vercel (see Setup Checklist below).

2. Calculator HTML Changes

Each of the 5 calculator HTML files received two small changes:

  • Phone gate: When the member unlocks the calculator, their phone number is saved in memory (not in browser storage) so it can be used later.
  • PDF button: When the member clicks "Download PDF", the calculator silently sends their phone + calculator name to the Vercel endpoint. This never blocks or delays the PDF download.
Privacy note: The phone number is held only in the page's JavaScript memory — it is never written to browser storage (localStorage or sessionStorage). It disappears the moment the tab is closed.
