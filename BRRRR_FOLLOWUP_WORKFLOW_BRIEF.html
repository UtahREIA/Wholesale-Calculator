BRRRR Calculator Follow-Up Automation

Behavior-triggered nurture sequence — fires when a member unlocks the BRRRR calculator via the phone gate
Date: April 14, 2026 Status: Code Complete — GHL Workflow Built, URLs Pending Trigger: Phone Gate Access

Overview

When a Utah REIA member unlocks the BRRRR calculator by entering their phone number, an automated follow-up sequence fires in GoHighLevel. The sequence connects them with the three most important resources for executing a BRRRR deal — refinance help, vetted contractors, and next-step education — delivered over 5 days in both SMS and email.

The logic in plain English:
Member unlocks BRRRR calculator → GHL identifies who they are by phone number → sends a 4-step resource sequence over 5 days (SMS + Email each step).

How It Works — Full Chain

CALCULATOR
Member enters their phone number into the BRRRR calculator phone gate and clicks "Unlock Calculator"
CALCULATOR
Phone is verified against Airtable. On success, the calculator fires a silent background POST request to the Vercel API — the member never sees this happen and there is no delay
VERCEL API
The /api/track-event endpoint receives the payload and forwards it to the GHL inbound webhook URL stored as an environment variable
{ "phone": "8015551234", "calculator": "BRRRR", "event": "calculator_access", "timestamp": "2026-04-14T18:30:00Z" }
GHL
GHL inbound webhook triggers the "BRRRR Access — Resource Nurture" workflow
GHL
Find Contact action looks up the CRM contact whose phone matches {{trigger.phone}} — this is what makes {{contact.first_name}} work in all messages
MESSAGES
4-step SMS + Email sequence fires over 5 days — each step tailored to a specific BRRRR resource (see message content below)
Important: The Find Contact step in GHL is what connects the phone number to a real contact record. Without it, GHL cannot personalise the messages with the member's name. This step has already been added to the workflow.

Why {{contact.first_name}} Works

The webhook only sends a phone number. GHL on its own cannot personalise messages from just a phone number — it needs to know which contact record that phone belongs to.

Without Find Contact

GHL receives the phone number but has no contact context.

{{contact.first_name}} renders blank.

Messages go out with no name.
With Find Contact ✓

GHL looks up the contact by phone in the CRM.

{{contact.first_name}} renders correctly.

Messages are fully personalised.
